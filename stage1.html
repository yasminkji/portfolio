Contexte de l'entreprise

SHOEMAKERS Group est une entreprise vendéenne spécialisée dans la fabrication et la commercialisation de chaussures pour enfants auprès de revendeurs.

Pour répondre à ses besoins commerciaux, l'entreprise exploite une application web B2B qui permet à ses revendeurs de consulter le catalogue et passer des commandes. Cette plateforme est un outil central dans la relation client.

Au moment de mon stage, l'application reposait sur Bonfire CI, un framework PHP construit au-dessus de CodeIgniter, dont la maintenance était devenue problématique : documentation rare, dépendances obsolètes et évolution technologique bloquée.

De plus, elle était hebergé sur un serveur web sous Debian 8, un serveur trop ancien pour garantir la sécurité et les performances de cette dernière.

L'entreprise souhaitait donc un Rétro-engineering complet de l'application pour faciliter l'ajout de nouvelles fonctionnalités à l'avenir.

Enjeux du stage

Comprendre l'existant

Analyser une application sans documentation, en explorant le code source et la structure de la base de données.

Documenter pour préparer la suite

Produire une cartographie claire de l'application existante.

Moderniser l'environnement

Configurer un serveur web à jour et adapté pour accueillir l'application et ses éventuelles évolutions.

Autonomie & méthode

Mener une mission d'analyse en grande autonomie, avec une démarche méthodique et professionnelle.

Déroulement des missions

1

Prise en main et exploration du projet existant

La première étape a consisté à installer et faire tourner l'application Bonfire CI localement, puis à explorer l'arborescence du code source : routes, contrôleurs, modèles, vues. J'ai cartographié les fonctionnalités existantes (catalogue, gestion des commandes, comptes revendeurs, tarification) pour obtenir une vision globale du système avant d'aller plus loin.

2

Analyse de la base de données

J'ai étudié le schéma de la base de données MySQL afin de comprendre les relations entre les entités (produits, clients, commandes, familles tarifaires…).

3

Rédaction du Cahier des Charges

À partir de l'analyse effectuée et des demandes des commerciaux, j'ai rédigé un Cahier des Charges des évolutions futures de l'application. Ce document recense les fonctionnalités actuelles à conserver, les dysfonctionnements identifiés, les évolutions souhaitées par l'entreprise, ainsi que les contraintes techniques à respecter lors de la migration envisagée.

4

Configuration d'un serveur web moderne

En parallèle, j'ai configuré un serveur web sous Linux (Apache, PHP 8+, MySQL) en vue d'accueillir l'application. J'ai veillé à respecter les bonnes pratiques de configuration (virtual hosts, permissions, variables d'environnement) afin de disposer d'un environnement stable et sécurisé pour la suite du projet.
